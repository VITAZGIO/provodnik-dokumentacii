# Проводник документации

Программа для ПК: папки заявок, раскладка файлов по подпапкам и приём фото с телефона.
Один файл `provodnik.exe` (~110 КБ) — ни установки, ни прав администратора, ни Python.
На телефоне ставить тоже ничего не нужно: только камера и браузер.

## Как пользоваться

Запустите `provodnik.exe` — откроется окно программы. При первом запуске нажмите
«Выбрать…» и укажите рабочую папку: ту, где лежат папки заявок. Она запомнится.

**Заявки.** Номер и наименование → «Создать папки». Появится папка `«номер - наименование»`
с подпапками и колонки под каждую. К старой заявке можно вернуться через «Существующие
заявки». Заявка ищется **по номеру** — это часть имени папки до « - », поэтому
наименование можно не помнить.

**Файлы.** Перетащите их (или целые папки) из Проводника в нужную колонку — они
скопируются, оригиналы останутся на месте. Совпадающие имена получают « (1)», « (2)»,
ничего не затирается. Кнопка «Добавить файлы» делает то же через обычный выбор файлов.
Клавиша Delete удаляет выбранное из папки заявки.

**Телефон.** Наведите камеру на QR-код в разделе «Приём с телефона». Откроется страница
отправки: номер заявки, категория, «Снять фото» → «Отправить». Файлы лягут прямо в папку
заявки на компьютере и сразу появятся в колонках — перетаскивать ничего не нужно.

Крестик прячет окно в значок рядом с часами — приём файлов с телефона продолжается.
Вернуть окно: двойной клик по значку или по `provodnik.exe`. Полностью выключить —
кнопка «Выключить сервер» внизу окна или тот же пункт в меню значка.

При первом запуске Windows один раз спросит про брандмауэр — нужно нажать
**«Разрешить доступ»** (частные сети), иначе телефон не достучится до ПК.

Настройки (рабочая папка, подпапки, ручной IP) хранятся в
`%USERPROFILE%\provodnik_config.json` — том же файле, что и у прежней Python-версии.

### Если QR не открывается на телефоне

Телефон должен быть в той же Wi-Fi сети, что и компьютер. Если сеть та же, а страница
не открывается — программа могла выбрать не тот сетевой адрес: у ПК их бывает несколько
(Docker, VPN, виртуальные машины). Под QR-кодом есть поле с найденными адресами —
выберите тот, который видит телефон, и нажмите «Сохранить». «Авто» вернёт автоопределение.

## Сборка

Исходники — в `.claude/provodnik/`. Собрать: двойной клик по `.claude/provodnik/build.bat`,
готовый exe появится в корне.

Visual Studio и .NET SDK не нужны: сборка идёт компилятором `csc.exe`, который уже входит
в состав Windows (.NET Framework 4.8 предустановлен во всех Windows 10 от версии 1903
и во всех Windows 11).

Иконка перерисовывается через `tools/MakeIcons.cs` — нужно редко, готовая лежит в `icons/`.

| файл                  | что делает                                                |
|-----------------------|-----------------------------------------------------------|
| `Program.cs`          | запуск, одна копия программы                              |
| `MainForm.cs`         | главное окно: заявки, колонки, QR-код                     |
| `ColumnPanel.cs`      | колонка подпапки с приёмом перетаскивания                 |
| `SubfoldersDialog.cs` | настройка списка подпапок                                 |
| `TrayApp.cs`          | значок рядом с часами                                     |
| `Files.cs`            | папки заявок: поиск, создание, копирование, удаление      |
| `Server.cs`           | приём файлов с телефона                                   |
| `Qr.cs`               | генератор QR-кода (свой, без внешних библиотек)           |
| `NetInfo.cs`          | определение локального IP                                 |
| `Config.cs`           | чтение и запись настроек                                  |
| `Util.cs`             | имена файлов, поиск папки заявки, JSON                    |
| `Theme.cs`            | палитра и шрифты                                          |
| `Pages.cs`, `web/`    | страница телефона                                         |

## Как это устроено внутри

Обычное оконное приложение (WinForms) плюс маленький HTTP-сервер, нужный только телефону.
Все операции с файлами делает сама программа, поэтому нет ни запросов разрешений на папку,
ни ограничений браузера.

Сервер написан на голом `TcpListener`, а не на `HttpListener`: тот работает через
системную службу и требует прав администратора на открытие порта.

Порт 8765, при занятости берётся следующий свободный до 8775. Страница телефона и приём
файлов защищены одноразовым кодом в ссылке: он случайный, меняется при каждом запуске,
без него отдаётся 403. Файл пишется во временный `.part` и переименовывается только
целиком, так что оборванная передача не оставляет битых файлов; недокачанные `.part`
удаляются при выходе.
