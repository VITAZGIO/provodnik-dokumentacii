# Проводник документации — сервер связи с телефоном

Программа для ПК: принимает фото заявок с телефона и сразу раскладывает их по папкам
заявок. На телефоне ставить ничего не нужно — только камера и браузер.

## Две версии

| файл                | что умеет                                                        | размер |
|---------------------|------------------------------------------------------------------|--------|
| `provodnik.exe`     | телефон и ПК в одной Wi-Fi сети                                   | ~100 КБ |
| `provodnik-pro.exe` | то же плюс «Публичный QR-код» — работает из любой сети, через интернет | ~52 МБ |

Обе — самостоятельные: скопировали на компьютер, запустили, всё. Ни установки, ни прав
администратора, ни Python, ни .NET-рантайма. Разница в размере из-за того, что внутрь
«pro» вшит `cloudflared` (утилита Cloudflare для туннеля).

Если сеть на работе блокирует туннель — берите обычный `provodnik.exe`, он там всё равно
единственный рабочий вариант.

## Как пользоваться

При первом запуске программа спросит рабочую папку — ту, где лежат папки заявок.
Дальше в окне будет QR-код: наводите на него камеру телефона и попадаете на страницу
отправки фото. Номер заявки → категория → «Снять фото» → «Отправить».

Файл попадает в папку заявки, найденную **по номеру** — это часть имени папки до « - ».
Если такой папки нет, создаётся новая `«<номер> - <наименование>»`. Внутри — подпапка
категории. Совпадающие имена получают « (1)», « (2)» — ничего не затирается.

Крестик прячет окно в трей (значок рядом с часами) — приём файлов продолжается.
Полностью выключить — кнопка «Выключить сервер» внизу окна или «Выход» в меню значка.
Если запустить exe, когда программа уже работает, — просто откроется её окно.

При первом запуске Windows один раз спросит про брандмауэр — нужно нажать
**«Разрешить доступ»** (частные сети), иначе телефон не достучится до ПК.

Настройки (рабочая папка, категории, ручной IP) хранятся в
`%USERPROFILE%\provodnik_config.json` — том же файле, что и у прежней Python-версии.

### Если QR не открывается на телефоне

Скорее всего, программа выбрала не тот сетевой адрес: у ПК их бывает несколько
(Docker, VPN, виртуальные машины). В окне есть список найденных адресов — выберите тот,
который видит телефон, и нажмите «Сохранить». «Авто» возвращает автоопределение.

### Публичный доступ (только `provodnik-pro.exe`)

Нужен, когда телефон и ПК в разных сетях. Кнопка «Публичный QR-код» → предупреждение →
набрать слово «ПОДТВЕРДИТЬ» → рядом появится второй QR с таймером. Выключится сам через
час. Ограничения: файл не больше 100 МБ (это лимит Cloudflare), файлы идут через серверы
Cloudflare, а адрес открыт любому, кто его узнает, — поэтому и подтверждение.

Туннелю нужен **исходящий порт 7844**. Корпоративные сети, антивирусы и VPN его часто
режут; тогда программа так и напишет, а QR показывать не станет — иначе телефон получил
бы ошибку 530. Сменить этот порт нельзя: он зашит в протокол Cloudflare.

## Исходники и сборка

Всё лежит в `.claude/provodnik/`. Собрать обе версии — двойной клик по
`.claude/provodnik/build.bat`, готовые exe появятся в корне.

Visual Studio и .NET SDK не нужны: сборка идёт компилятором `csc.exe`, который уже
входит в состав Windows (.NET Framework 4.8 предустановлен во всех Windows 10 от версии
1903 и во всех Windows 11).

Для сборки «pro» нужен `cloudflared.exe` рядом с `build.bat` — в репозиторий он не
кладётся (52 МБ чужого бинарника). Взять
[здесь](https://github.com/cloudflare/cloudflared/releases/latest), файл
`cloudflared-windows-amd64.exe`, переименовать в `cloudflared.exe`. Без него соберётся
только обычная версия.

Иконки перерисовываются через `.claude/provodnik/tools/MakeIcons.cs` — нужны редко,
готовые `.ico` лежат в `icons/`.

| файл               | что делает                                                   |
|--------------------|--------------------------------------------------------------|
| `Program.cs`       | запуск, серверы, одна копия программы                        |
| `MainForm.cs`      | окно с QR-кодами, трей                                       |
| `ConfirmDialog.cs` | предупреждение перед публичным доступом                      |
| `Server.cs`        | HTTP-сервер и приём файлов                                   |
| `Qr.cs`            | генератор QR-кода (свой, без внешних библиотек)              |
| `NetInfo.cs`       | определение локального IP                                    |
| `Config.cs`        | чтение и запись настроек                                     |
| `Util.cs`          | имена файлов, поиск папки заявки, JSON                       |
| `Tunnel.cs`        | запуск cloudflared                                           |
| `Pages.cs`, `web/` | страница телефона                                            |

Обе версии собираются из одного исходника: обычная — с ключом `LOCAL_ONLY`, который
отключает всё, что связано с туннелем.

## Как это устроено внутри

Сервер написан на голом `TcpListener`, а не на `HttpListener` — тот работает через
системную службу и требует прав администратора на открытие порта.

Доступ защищён одноразовым кодом в ссылке: он случайный, меняется при каждом запуске,
без него страница и загрузка отдают 403.

В «pro» поднимается **два** сервера: основной для локальной сети и отдельный на
`127.0.0.1` — только для cloudflared. Так сделано не для красоты: cloudflared работает
на этом же ПК, и все запросы из интернета приходят к нам с локального адреса, поэтому
«свой/чужой» по IP отличить невозможно. Решает то, в какую дверь пришёл запрос — это
не подделать. На интернет-двери и лимит другой: 100 МБ вместо 500.
